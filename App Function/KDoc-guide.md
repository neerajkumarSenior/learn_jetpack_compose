# 📝 AppFunctions & KDoc: How AI Understands Your Code

इस डॉक्यूमेंट में यह समझाया गया है कि **Google I/O '26** के नए आर्किटेक्चर में **KDoc (Kotlin Documentation)** का उपयोग कैसे किया जाता है और क्यों यह सिर्फ "कमेंट्स" नहीं, बल्कि AI एजेंट के लिए **"प्रॉम्ट और रूल्स"** का काम करते हैं।

---

## 🧐 AppFunctions में KDoc क्यों महत्वपूर्ण है?

ट्रेडिशनल एंड्रॉइड डेवलपमेंट में, KDoc का उपयोग केवल डेवलपर्स के लिए कोड को समझने के लिए किया जाता था। लेकिन **AppFunctions (Model Context Protocol)** के साथ:

1. **AI Semantic Mapping:** जब आप प्रोजेक्ट को कंपाइल करते हैं, तो AppFunctions का कंपाइलर आपके KDoc कमेंट्स को एक्सट्रैक्ट करके एक **JSON Schema** बना देता है।
2. **Intent Matching:** जब यूजर AI (जैसे Gemini) से कुछ कहता है, तो AI इस JSON Schema (यानी आपके लिखे KDoc) को पढ़कर तय करता है कि यूजर की बात को पूरा करने के लिए आपके ऐप का कौन सा फ़ंक्शन सबसे सही है।
3. **Parameter Parsing:** KDoc में लिखे `@param` को देखकर AI समझ जाता है कि यूजर की बोली गई भाषा (Natural Language) में से कौन सा शब्द किस वेरिएबल (`amount`, `category`) में भेजना है।

---

## 🛠️ KDoc लिखने का सही तरीका (Best Practices & Syntax)

एक अच्छे AppFunction के लिए KDoc लिखते समय आपको **3 मुख्य बातें** ध्यान में रखनी होती हैं:
* **Function Summary:** फ़ंक्शन क्या करता है, इसे साफ और स्पष्ट शब्दों में लिखें।
* `@param` **Descriptions:** हर पैरामीटर का क्या मतलब है और AI को उसमें किस तरह का डेटा (जैसे रुपए, नाम, तारीख) ढूंढना है, यह साफ बताएं।
* **Language Support:** आप इसे इंग्लिश या हिंदी दोनों में लिख सकते हैं, AI मॉडल (LLM) इसे आसानी से समझ लेते हैं।

### 💻 Code Example: Expense Tracker Function

```kotlin
/**
 * यूजर के द्वारा किए गए खर्चों को ट्रैक करके सीधे लोकल डेटाबेस में सेव करता है।
 * इस फ़ंक्शन का उपयोग तब करें जब यूजर अपने पैसों के खर्च, बिल पेमेंट, या शॉपिंग का रिकॉर्ड रखना चाहे।
 * 
 * @param amount खर्च की गई कुल राशि या रकम (हमेशा डबल/डेसिमल फ़ॉर्मेट में होनी चाहिए, जैसे 150.0)
 * @param category खर्च की श्रेणी या केटेगरी (उदा. Food, Transport, Shopping, Medical)
 * @return सफलता या विफलता का संदेश जो सीधे यूजर को स्क्रीन पर दिखाया जाएगा
 */
@AppFunction
suspend fun addExpenseFromAI(amount: Double, category: String): String {
    // Business Logic...
    return "सफलतापूर्वक ₹amount को 'category' केटेगरी में जोड़ दिया गया है!"
}
```

## 🔄 Behind the Scenes: KDoc से JSON Schema कैसे बनता है?

जब आप ऐप को बिल्ड करते हैं, तो `appfunctions-compiler` बैकग्राउंड में आपके KDoc को एक ऐसी भाषा में बदल देता है जिसे AI एजेंट समझ सके। 

ऊपर लिखे KDoc का बैकग्राउंड स्ट्रक्चर कुछ इस तरह दिखता है:

```json
{
  "functionName": "addExpenseFromAI",
  "description": "यूजर के द्वारा किए गए खर्चों को ट्रैक करके सीधे लोकल डेटाबेस में सेव करता है...",
  "parameters": {
    "amount": {
      "type": "number",
      "description": "खर्च की गई कुल राशि या रकम (हमेशा डबल/डेसिमल फ़ॉर्मेट में होनी चाहिए)"
    },
    "category": {
      "type": "string",
      "description": "खर्च की श्रेणी या केटेगरी (उदा. Food, Transport, Shopping)"
    }
  }
}
```

## 🛑 Common Mistakes to Avoid (क्या न करें)

❌ **अधूरा KDoc:** `@param` के आगे कुछ न लिखना। इससे AI कंफ्यूज हो जाएगा कि उस वेरिएबल में क्या डेटा पास करना है।
❌ **टेक्निकल शब्दों का ज्यादा इस्तेमाल:** KDoc में *"यह फ़ंक्शन SQL queries चलाता है"* ऐसा न लिखें। AI को आसान भाषा में बताएं कि यह यूजर के लिए क्या काम करता है (जैसे: *"यह खर्चों को सेव करता है"*).
